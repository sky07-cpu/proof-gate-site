# Proof Gate 紹介サイト

生成AI／RAG型チャットボットの回答品質を、テストケースで検査し、人間が承認し、その記録を残す品質管理基盤「Proof Gate」の紹介用ランディングページです。

現在は見た目と情報設計を確認するための、静的な1ページサイトです。ログイン、データベース、API連携、問い合わせフォームの送信処理は実装していません。

## 使用技術

- Vite
- React
- TypeScript
- 通常のCSS（Tailwind CSSや外部UIライブラリは不使用）

## 各技術の役割

| 技術 | このプロジェクトでの役割 |
| --- | --- |
| HTML | ページの土台です。`index.html` にReactを表示するための入口となる要素を置いています。 |
| CSS | 色、余白、文字サイズ、カード、レスポンシブ表示など、見た目を担当します。 |
| JavaScript | ブラウザ上で表示や操作を動かす言語です。このプロジェクトでは、より安全に書けるTypeScriptを経由してJavaScriptに変換されます。 |
| TypeScript | JavaScriptに「型」を加えた言語です。データの間違いに開発中に気づきやすくし、将来の機能追加を安全にします。 |
| React | 画面を部品とデータから組み立てるライブラリです。カードの繰り返し表示などを分かりやすく記述しています。 |
| Vite | 開発サーバーの起動と、本番公開用ファイルの生成を担当する開発ツールです。 |
| Node.js | ViteやTypeScriptなどの開発ツールをPC上で動かす実行環境です。 |
| npm | プロジェクトで使うパッケージのインストールと、開発・ビルド用コマンドの実行を担当します。 |

## ファイル構成

```text
proof-gate-site/
├─ public/
│  └─ favicon.svg       # ブラウザのタブに表示するアイコン
├─ src/
│  ├─ App.tsx           # LP全体の内容と画面構成
│  ├─ main.tsx          # ReactをHTMLへ読み込む入口
│  ├─ styles.css        # 全体のデザインとレスポンシブ対応
│  └─ vite-env.d.ts     # Vite用のTypeScript型定義
├─ index.html           # ブラウザが最初に読むHTML
├─ package.json         # 使用パッケージとnpmコマンドの定義
├─ tsconfig.json        # TypeScript設定の入口
├─ tsconfig.app.json    # Web画面向けのTypeScript設定
├─ tsconfig.node.json   # Vite設定ファイル向けのTypeScript設定
└─ vite.config.ts       # ViteとReactをつなぐ設定
```

初心者でも全体を追いやすいように、ページ本体は `App.tsx`、見た目は `styles.css` にまとめています。ページが大きくなった段階で、セクション単位のコンポーネントへ分割できます。

## 今回追加した動き

このサイトは静的な1ページLPですが、企業サイトらしく見えるように、最小限の動きを入れています。

- FAQの開閉
  - 質問をクリックすると回答が開いたり閉じたりします。
  - `App.tsx` の `openFaqIndex` というReact stateで「今どのFAQが開いているか」を管理しています。
- スマホ用ナビゲーション
  - 画面幅が狭いときはハンバーガーメニューを表示します。
  - `App.tsx` の `isMenuOpen` というReact stateで「メニューが開いているか」を管理しています。
- スクロール時のふわっと表示
  - セクションやカードが画面に入ったタイミングで、控えめに表示されます。
  - `IntersectionObserver` というブラウザ機能で表示タイミングを検知し、CSSのクラスを付け替えています。
- ホバーアニメーション
  - CTAボタンやカードにマウスを乗せると、少し浮くような動きをします。
  - これは主に `styles.css` の `transition` と `transform` で実装しています。

React stateは「画面の状態を覚えておく箱」のようなものです。今回であれば、FAQやメニューのように、ユーザー操作によって表示が変わる部分にだけ使っています。

## ローカルで動かす手順

### 1. 必要なもの

- Node.js 20.19以上（または22.12以上）
- npm（Node.jsと一緒にインストールされます）

### 2. パッケージをインストール

ターミナルでこのプロジェクトのフォルダを開き、次を実行します。

```bash
npm install
```

### 3. 開発サーバーを起動

```bash
npm run dev
```

ターミナルに表示されたURL（通常は `http://localhost:5173`）をブラウザで開きます。ファイルを保存すると、表示が自動で更新されます。

### 4. 開発サーバーを止める

ターミナルで `Ctrl + C` を押します。

## 本番公開用ファイルを作る

```bash
npm run build
```

問題がなければ `dist` フォルダに、公開用のHTML・CSS・JavaScriptが生成されます。

生成結果をローカルで確認する場合は次を実行します。

```bash
npm run preview
```

## 現在の実装範囲

- 1ページの企業向けランディングページ
- PC・タブレット・スマートフォン向けレスポンシブ表示
- 導入対象、機能詳細、料金プラン、FAQ、フッター
- ページ内リンクによるCTA
- FAQの開閉
- スマホ用ハンバーガーメニュー
- スクロール時の表示演出
- CTAボタンやカードのホバーアニメーション
- CSSで作成した製品画面イメージ

問い合わせボタンは見た目確認用です。実際の送信先やフォーム処理は、将来のWebアプリ化・運用要件に合わせて追加してください。
